# Changelog

## v0.1.5 (2026-06-28)

### ✨ 优化
- **Bark 通知摘要**：按「已看完 / 在看 / 失败 / 跳过」分组展示具体片名，失败与跳过项目附带原因
- **通知长度控制**：每类最多展示 3 个名称，超出部分显示剩余数量，避免批量同步时通知过长

### ✅ 测试
- 增加 Bark 多状态摘要和长列表折叠的回归测试

## v0.1.4 (2026-06-28)

### 🔧 修复
- **飞牛电影分类**：兼容电影的季号、集号为 `0` 且父级为空字符串的数据结构，避免误判为电视剧后被静默跳过

### ✅ 测试
- 增加飞牛电影零值字段与正常电视剧单集的分类回归测试

## v0.1.3 (2026-06-27)

### 🔧 修复
- **已观看影片进度识别**：飞牛影视已标记「已观看」时按 100% 处理，避免播放位置归零导致影片被误判为 0%
- **播放记录变化检测**：将已观看状态纳入指纹，确保状态变化后能够重新触发同步

### ✅ 测试
- 增加已观看标记优先级、常规播放进度及状态指纹的回归测试

## v0.1.2 (2026-06-22)

### 🔧 修复
- **FNTV 数据库路径**：docker-compose 默认路径修正为飞牛 OS 真实路径（`/usr/local/apps/@appdata/trim.media/database/trimmedia.db`），替代之前的符号链接

## v0.1.1 (2026-06-22)

### ⚙️ 基础设施
- **GitHub Actions CI**：自动构建并推送 Docker 镜像到 Docker Hub

### 📖 文档
- 播放阈值说明改为"支持自定义"，消除"固定 90%"的歧义
- README 功能表格增加 Bark 推送通知说明

## v0.1.0 (2026-06-22)

### ✨ 新功能
- **Bark 推送通知**：同步完成后推送结果到 iOS，Cookie 过期时主动告警
- **Cron 定时调度**：支持间隔模式和 Cron 表达式两种调度方式
- **播放记录指纹检查**：播放记录无变化时跳过豆瓣 API 调用，节省配额
- **CookieCloud 集成**：支持通过 CookieCloud 同步豆瓣 Cookie
- **SSE 实时日志**：浏览器端流式展示同步过程
- **智能跨季推断**：自动识别飞牛扁平结构中的季度边界

### 🎨 界面
- 全局 CSS 主题：侧边栏布局、卡片式分组、响应式适配
- 配置页重构为手风琴式卡片组
- 状态面板信息聚合，实时日志展示
- 同步日志增加分页和全文搜索
- Cookie 提取图文指南页面

### 🔧 修复
- 同步状态卡在"同步中"的问题
- 历史页翻页阈值错误
- 豆瓣搜索兼容多种链接格式（直接链接 / 跟踪链接 / onclick）
- Cookie 认证 403 自动刷新 ck 后重试

### 📖 文档
- README 全面翻新：徽章栏、截图画廊、架构图、功能表格
- 同步规则改写为中文白话描述
- 增加 Docker 部署和配置指南
