# Changelog

## v0.1.2 (2026-06-22)

- **FNTV 数据库路径**：docker-compose 默认路径修正为飞牛 OS 真实路径（`/usr/local/apps/@appdata/trim.media/database/trimmedia.db`），替代之前的符号链接

## v0.1.1 (2026-06-22)

### ⚙️ 基础设施
- **GitHub Actions CI**：自动构建并推送 Docker 镜像到 Docker Hub

### 📖 文档
- 播放阈值说明改为"支持自定义"，消除"固定 90%"的歧义
- README 功能表格增加 Bark 推送通知说明

## v0.1.0 (2026-06-22)

### ✨ 新功能
- **Bark 推送通知**：同步完成后推送结果到 iOS，Cookie 过期时主动告警
- **Cron 定时调度**：支持间隔模式和 Cron 表达式两种调度方式
- **播放记录指纹检查**：播放记录无变化时跳过豆瓣 API 调用，节省配额
- **CookieCloud 集成**：支持通过 CookieCloud 同步豆瓣 Cookie
- **SSE 实时日志**：浏览器端流式展示同步过程
- **智能跨季推断**：自动识别飞牛扁平结构中的季度边界

### 🎨 界面
- 全局 CSS 主题：侧边栏布局、卡片式分组、响应式适配
- 配置页重构为手风琴式卡片组
- 状态面板信息聚合，实时日志展示
- 同步日志增加分页和全文搜索
- Cookie 提取图文指南页面

### 🔧 修复
- 同步状态卡在"同步中"的问题
- 历史页翻页阈值错误
- 豆瓣搜索兼容多种链接格式（直接链接 / 跟踪链接 / onclick）
- Cookie 认证 403 自动刷新 ck 后重试

### 📖 文档
- README 全面翻新：徽章栏、截图画廊、架构图、功能表格
- 同步规则改写为中文白话描述
- 增加 Docker 部署和配置指南
